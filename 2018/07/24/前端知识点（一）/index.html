前端知识点（一）

作死开坑

  • 构造函数中用var声明的变量不会绑定给实例，作为构造函数的局部变量存在
  • 当全局变量与函数形参同名时，在函数内部改变变量值时，无论是var重新声明还是直接 变量＝xx 都不会覆盖全局变量的值；但如果该全局变量是引用类型(对象等，例如A＝{n:2})，因为内存地址是一样的，所以会改变
    函数形参相当于一个特殊的局部变量
    而如果全局变量与形参不同名，但函数中出现全局变量的赋值时，会改变
  • 两个同名的函数的声明，后者会覆盖前者，再加上声明提升，所以一开始就是后一个函数生效
  • var a＝b＝1;相当于var a＝1;b＝1;这样一来，b就相当于是全局变量
  • 1＋ ＋”2”＝3因为＋”2”中的加号为一元操作符，相当于Number()；类似的还有!!强制转boolean，””＋强制转String
    1＋”2”＝12因为字符串加数字相当于字符串拼接
    NaN＋任何数字都是NaN，但是可以与字符串拼接
  • spa，ssr：spa就是单页应用，饿了么例子就是；ssr在前端就是服务器渲染，vue有专门的框架实现，要配置webpack的server entry；ssr在网络方面是一种翻墙手段
  • dom：文档对象模型，给html和xml文件使用的一组api，包括属性，方法和事件
  • navigator.userAgent用于获取浏览器类型
  • 根据参数新建Date对象实例，减去默认Date对象实例，得出秒数，进而算出剩余时间
  • flexible手淘框架的核心原理就是根据不同的width给网页中html跟节点设置不同的font-size，然后所有的距离大小都用rem来代替，这样就实现了不同大小的屏幕都适应相同的样式了
    flexible：移动端布局方案，实质是动态改写meta标签和font-size以及data-dpr，还将px转换成rem
  • 同源策略：不同域的脚本不会被执行
  • 箭头函数中的this是在定义函数的时候绑定，普通函数的this则是在调用时绑定
    箭头函数不能作为构造函数，因为不能new
    没有arguments对象，要用…rest参数（不定参）实现
  • 前端优化：减少http请求次数，减少dom操作，优化选择器，合理编码与优化代码
  • v-if动态添加dom，v-show切换display；前者切换代价高，后者初始化慢
  • 逗号操作符 对它的每个操作对象求值（从左至右），然后返回最后一个操作对象的值
  • 垂直居中：flex布局的align-items:center
    或 display:table-cell＋vertical-align:middle
    或 absolute＋left和top:50% 然后设置负margin （需知宽高）
    或 position:fixed/absolute＋margin:auto
    或 position:absolute＋top:50%＋
    transform:translateY(-50%)
  • vertical-align适用于行内标签和display:table-cell；text-align适用于display:block
  • react和vue的区别：前者jsx后者html；前者必须状态管理state后者将数据置于data对象中；前者适合大型应用后者小型应用 前者还可以开发原生应用
  • vue路由：url改变，通过hashchange事件监听改变，通知给vue实例，再根据当前地址重新渲染router-view
  • vuex中只有通过mutation才能改变state，有点像事件注册，但必须是同步函数；action用于触发mutation，可以在action内部执行异步操作，用this.$store.dispatch分发action，也可用mapAction辅助函数
  • 圣杯布局：左右固定宽度，中间100%；中间dom先写，两边后写，左边左margin-100%，右边左margin-自己宽度；父元素设置左右padding，左右块移过去
    双飞翼布局：效果和圣杯一样，区别是中间栏的内容不被遮挡的方式不一样，双飞翼直接创建子div放内容，设置margin为两侧留出空位
    如果用flex布局的话，就是flex-grow和flex-basic的配合
  • http缓存：服务器端的缓存，服务器发送缓存规则和内容给浏览器，浏览器保存；
    分为强制缓存与对比缓存；
    前者如果有相应缓存就不请求服务器，后者无论缓存是否命中，都会向服务器发送缓存验证标识，来判断是否失效（存在）；
    两者可同时存在，前者优先级高；
    缓存的规则信息都在http header中；用header中的Cache-Control控制缓存规则，etag/if-none-match或者last-modified/if-modified-since判断是否改变
  • weex（阿里）：使用web开发体验来开发高性能原生应用的框架，可以配合vuejs来开发原生app；原理是把vue写出来的web页面打包成一个js包，通过特殊的js引擎转换成原生应用
  • 发布订阅模式：
    1.确认一个发布者
    2.给发布者添加一个列表用于存放订阅者和回调函数
    3.发布：发布者遍历列表，按订阅者触发回调函数
    4.订阅：向列表中添加订阅者身份标识和回调函数
  • form标签：用entype属性规定表单提交前 内容的编码形式，默认www-form编码，有上传文件时要multipart/form-data编码
  • 渐进增强/优雅降级：先上兼容/向下兼容；在css中就是前者先写兼容css照顾低版本浏览器再写正常css，后者相反；这两个只是看待同种事物的两种观点
  • retina屏幕：像素每英寸（ppi）大于300的超高像素密度的屏幕；
    dpr：设备像素比，可以用来表示屏幕的等级，dpr＝2意味着css中1px对应2px物理像素
    配合viewport知识点食用更佳
  • @dpclick：vue 双击事件
  • js原生弹窗：alert/confirm（确认框）/prompt（输入提示框）
  • es6…展开运算符：可以在不定形参时用…arg代替；解构赋值；将类数组对象变成数组对象；展开运算符一般在最后一个
    不定参还可以直接用arguments对象来实现，通过判断arguments的长度来确定有几个参数
    不定参可实现js函数的重载
  • 深拷贝：递归拷贝，判断子内容是不是对象决定是否递归和继续
  • 带cookie的跨域也用CORS解决，只是对相应头的设置要改变
  • 匿名函数的this指向window，因为故意设计成window是匿名函数的调用者
  • js中的垃圾回收机制：一种是标记清除，将进入环境的变量标记为“进入环境”，执行结束后标记为“离开环境”，然后在触发回收间隔后清理回收；另一种是引用计数，每次被引用，变量计数就加一，如果不引用了就减一，当引用计数为0时，在触发回收间隔后就会清理回收掉；后者容易触发内存泄漏；
    内存泄漏：用不到的内存，没有及时释放就是内存泄漏
  • dom0级事件：行间click事件和js中onclick绑定的事件，后者覆盖前者；
    dom2级事件：addEventListener绑定的事件，可与dom0共存；
    dom3事件：在已有基础上添加一些新的事件，并为事件对象添加一些新的属性
  • 链式调用基本原理：return this
  • promise.all：将多个promise实例包装成一个实例，全部完成时包装后实例才为完成
    promise.race：将多个promise包装在一起，只要有一个完成就会改变包装后实例的状态
  • BFC：块级格式化上下文，页面中的具有一套渲染规则的渲染区域；容器里面的元素不会在布局上影响外面的元素；同一个BFC下外边距会折叠；float的值不为none，overflow不为visible，position为absolute或fixed，display的值为inline-block或table-cell就会形成BFC
  • URL和URI：前者统一资源定位符，后者统一资源标识符；URL包含URI
  • xss：跨站脚本攻击，相当于html注入，如果不过滤执行了js代码，可能导致cookie泄露等。防止：过滤
    csrf：跨站请求伪造（cookie），挟制用户在当前已登录的Web应用程序上执行非本意的操作。防止：设置token（存在本地，访问时带上）、验证码、写操作用post、JSON API禁用CORS、禁用跨域请求、检查referrer
  • css3 animation是纯css实现，而jQuery animation是快速切换css样式实现的；前者效率高，后者兼容性好
  • ajax不能前进后退的原因是因为ajax不用刷新页面就可以获取数据，这样window.history（历史记录）中就没有相关的URL保存记录；只要想办法解决history记录的问题就行：html5的pushState方法和添加location的hash都可以解决

  • require是CommonJS规范的一个全局性方法用于加载模块；exports是CommonJS的导出方法；export为es6的模块规定对外接口；export default为模块指定默认输出，在import的时候指定任意名字；import是es6的模块加载方法，用as关键字可以重新命名
    import和require的区别：前者动态引用，后者立即加载
    import和export最终都是编译成require和exports

  • AMD：异步(A)模块定义；
    定义：define(id?,dependencies?,factory)模块名称（字符串），依赖模块（数组形式），方法（模块函数）
    加载：require([module],callback);
    CMD：通常(C)模块定义；定义的方法与AMD一样；
    AMD与CMD的区别：AMD前置依赖，立即加载依赖模块；CMD就近依赖，要把模块变成字符串解析一遍才知道依赖哪些模块
  • webpack loader：通过不同的loader使webpack可以调用外部脚本或工具，实现对不同格式的文件的处理，如.vue对应vue-loader以及css预编译的转换
  • nodejs：用http模块创建了服务器后，用url模块处理get请求，用模块在请求头中获取post的数据，用mysql模块连接mysql，有events模块可以设置相应的事件回调函数，有process模块可以实现子线程，用fs模块实现文件读取写入
  • vue组件库（插件）：编写导出文件（install函数（vue插件必须要有这个函数）中注册全局组件，然后全局vue作为参数），webpack配置（加上external和library，这样才会打包vue），package.json配置
